Connect AI Tools

Paste your proxy URL into any MCP-compatible AI tool. You'll only see tools from servers you've connected above.

Claude Code (CLI)

Run once in your terminal:

claude mcp add --transport http mcp-proxy {{.ProxyURL}}

Or add manually to ~/.claude.json:

{
  "mcpServers": {
    "mcp-proxy": {
      "type": "http",
      "url": "{{.ProxyURL}}"
    }
  }
}
Claude Desktop

Edit your config file and restart Claude Desktop:

macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json

{
  "mcpServers": {
    "mcp-proxy": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "{{.ProxyURL}}", "--allow-http"]
    }
  }
}
Gemini CLI

Run once in your terminal:

gemini mcp add mcp-proxy {{.ProxyURL}}

Or add manually to ~/.gemini/settings.json:

{
  "mcpServers": {
    "mcp-proxy": {
      "httpUrl": "{{.ProxyURL}}"
    }
  }
}
Cursor

Create or edit .cursor/mcp.json in your project root:

{
  "mcpServers": {
    "mcp-proxy": {
      "type": "http",
      "url": "{{.ProxyURL}}"
    }
  }
}
VS Code (GitHub Copilot)

Create or edit .vscode/mcp.json in your project root:

{
  "servers": {
    "mcp-proxy": {
      "type": "http",
      "url": "{{.ProxyURL}}"
    }
  }
}
Windsurf

Open Windsurf Settings → MCP and add a new server, or edit ~/.codeium/windsurf/mcp_config.json:

{
  "mcpServers": {
    "mcp-proxy": {
      "serverUrl": "{{.ProxyURL}}"
    }
  }
}

Dual-Proxy Setup (local + company proxy)

If your company also runs a hosted proxy, configure your AI tool with both URLs to get tools from both sources:

Claude Code — run both commands:

claude mcp add --transport http mcp-proxy-local {{.ProxyURL}}
claude mcp add --transport http mcp-proxy-company https://YOUR_COMPANY_PROXY/mcp/YOUR_TOKEN

Replace YOUR_COMPANY_PROXY and YOUR_TOKEN with the values from your company's proxy dashboard.

Each proxy contributes its own tools. Adding a server to either proxy makes it immediately available in your AI tool.
