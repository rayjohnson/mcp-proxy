# Implementation Plan: Per-User Server Enable/Disable

**Branch**: `011-server-enable-disable` | **Date**: 2026-05-25 | **Spec**: [spec.md](spec.md)

## Summary

Add a per-user toggle to each connected server (both HTTP upstreams and stdio catalog entries) on the dashboard. Toggle state is persisted in the database. The session layer filters out disabled servers when opening new MCP sessions, so their tools are never exposed. Credentials and configuration are preserved; toggling is purely a visibility/connection control.

## Technical Context

**Language/Version**: Go 1.23

**Primary Dependencies**: stdlib only (no new third-party deps); existing `go-sdk/mcp`, SQLite via existing driver

**Storage**: SQLite (local mode) — add `enabled` column to `upstream_configs`, new `server_toggles` table for stdio catalog entries. PostgreSQL (cloud mode) — same schema changes via separate migration.

**Testing**: stdlib `testing`

**Target Platform**: macOS / Linux server (both local and cloud modes)

**Project Type**: Web service + MCP proxy

**Performance Goals**: Toggle API must respond in < 100ms (single DB write)

**Constraints**: No new third-party dependencies; no structural changes to existing credential storage

**Scale/Scope**: Single-user local mode primary; multi-user cloud mode supported
