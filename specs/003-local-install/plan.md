# Implementation Plan: Local Mode Packaging & Distribution

**Branch**: `003-local-install` | **Date**: 2026-05-24 | **Spec**: [spec.md](spec.md)

## Summary

Package the local-mode proxy for zero-friction macOS installation: a curl-installable shell script downloads a pre-built binary from GitHub Releases, stores an encryption key in the macOS Keychain, and registers a launchd LaunchAgent for auto-start on login. Default port changes from 8080 to 9753 to avoid conflicts with common dev tools. The SQLite backend gains a proper migration runner (PRAGMA user_version + go:embed) so schema changes survive binary upgrades without data loss.

## Key Design Decisions

### Port: 9753
Unassigned by IANA, not used by any common developer tool. The 9-7-5-3 descending pattern makes it memorable. See research.md.

### Keychain key storage
`go-keyring` writes to the macOS Keychain under service=`"mcp-proxy"`, account=`"encryption-key"`. The key is generated once at install time and never rotated by upgrades. The existing `LOCAL_KMS_KEY` env var remains as an override for `make run-local` and CI use.

### launchd bootstrap/bootout (not load/unload)
macOS 13+ deprecated `launchctl load/unload`. The install script uses `launchctl bootstrap gui/$(id -u)` and `launchctl bootout gui/$(id -u)` exclusively. The `gui/` domain is required for Keychain access.

### PRAGMA user_version migrations
Each numbered `.sql` file in `internal/store/sqlite/migrations/` corresponds to one schema version. The migration runner applies missing migrations in order inside individual transactions, updating `PRAGMA user_version` atomically per migration. `001_initial.sql` is the current schema verbatim. Future changes add new numbered files — no changes to existing migration files.

### Binary version via ldflags
GoReleaser injects `-X main.version={{.Version}}` at build time. `make run-local` and `go run` builds get `version=dev`. The `--version` flag prints `mcp-proxy <version>` so the install script can detect whether an upgrade is needed without downloading anything.

### Install script idempotency
If the installed version equals the latest release tag, the script exits 0 without downloading. The Keychain entry is only created if it doesn't already exist. The plist is rewritten on every upgrade run (to pick up any changed flags).
